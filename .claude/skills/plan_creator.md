# Skill: Plan Creator

## Description
Claude reasoning loop that analyzes tasks in Needs_Action and creates
detailed Plan.md files with step-by-step execution strategies.

## Trigger
- New task appears in `/Needs_Action/`
- Manual planning request
- Command: `/create-plan`, `/plan-task`

## Inputs
| Input | Type | Required | Description |
|-------|------|----------|-------------|
| task_path | String | Yes | Path to task file |
| planning_depth | String | No | quick, standard, detailed (default: standard) |
| constraints | Array | No | Constraints to consider |
| deadline | DateTime | No | Task deadline if any |

## Process

### Step 1: Task Analysis
1. Read task file from `/Needs_Action/`
2. Parse metadata and content
3. Identify task type and requirements
4. Assess complexity

### Step 2: Context Gathering
1. Check `/Company_Handbook.md` for relevant rules
2. Check `/Business_Goals.md` for alignment
3. Review similar past tasks in `/Done/`
4. Identify dependencies

### Step 3: Plan Generation
1. Define objective clearly
2. Break down into actionable steps
3. Identify required resources
4. Assess risks and mitigation
5. Set milestones

### Step 4: Plan Documentation
1. Create Plan.md file in `/Plans/`
2. Link to original task
3. Include validation checklist
4. Add approval section if needed

### Step 5: Task Update
1. Update original task status
2. Link plan to task
3. Log planning activity

## Output
| Output | Type | Description |
|--------|------|-------------|
| plan_path | String | Path to created plan file |
| steps_count | Integer | Number of steps in plan |
| requires_approval | Boolean | If plan needs human approval |
| estimated_complexity | String | low, medium, high |

## Plan Template

```markdown
---
type: plan
plan_id: PLAN_{task_id}_{timestamp}
related_task: {task_path}
created: {ISO_timestamp}
status: draft
complexity: {complexity}
requires_approval: {boolean}
---

# Plan: {task_title}

## Objective
{clear_objective_statement}

## Related Task
- **Task ID:** {task_id}
- **Location:** {task_path}
- **Priority:** {priority}

## Context
{relevant_context_from_handbook_and_goals}

## Execution Steps

### Phase 1: {phase_name}
| Step | Action | Owner | Status |
|------|--------|-------|--------|
| 1.1 | {action} | AI/Human | Pending |
| 1.2 | {action} | AI/Human | Pending |

### Phase 2: {phase_name}
| Step | Action | Owner | Status |
|------|--------|-------|--------|
| 2.1 | {action} | AI/Human | Pending |
| 2.2 | {action} | AI/Human | Pending |

## Resources Required
- {resource_1}
- {resource_2}

## Risks & Mitigation
| Risk | Likelihood | Mitigation |
|------|------------|------------|
| {risk} | {level} | {strategy} |

## Validation Checklist
- [ ] All steps are actionable
- [ ] Dependencies identified
- [ ] Risks assessed
- [ ] Aligned with business goals

## Approval
{if_requires_approval}
- [ ] Plan reviewed by human
- [ ] Approved for execution

---
*Generated by Plan Creator Skill*
```

## Planning Depth Levels

### Quick (5-10 min)
- High-level steps only
- Minimal risk assessment
- For simple, routine tasks

### Standard (15-30 min)
- Detailed step breakdown
- Basic risk assessment
- Resource identification
- For typical business tasks

### Detailed (30-60 min)
- Comprehensive analysis
- Full risk assessment
- Multiple scenarios
- For complex, high-impact tasks

## Example Usage

```
Skill: plan_creator
Input:
  task_path: "/Needs_Action/email/EMAIL_client_proposal.md"
  planning_depth: "standard"
  deadline: "2026-02-25"

Output:
  plan_path: "/Plans/PLAN_EMAIL_client_proposal_20260220.md"
  steps_count: 8
  requires_approval: true
  estimated_complexity: "medium"
```

## Integration with Other Skills
- Uses `approval_workflow` for sensitive plans
- Informs `gmail_watcher` for email-related tasks
- Updates `Dashboard.md` via orchestrator

## Safety Rules
- Plans for payments always require approval
- Plans affecting external parties need review
- Never execute without validated plan
- Log all planning decisions
